# Manifest Guide

## Overview

Manifests are YAML files that define which documents to ingest, along with their metadata. The Covenant Systems uses multiple manifests for different use cases.

---

## 📋 Available Manifests

| Manifest | Size | Docs | Purpose | Use When |
|----------|------|------|---------|----------|
| **manifest.yaml** | 20KB | All | Master registry (merged) | Production, full deployment |
| **milestone_1_fintrac_pcmltfa.yaml** | 16KB | 36 | Complete PCMLTFA framework | Building FINTRAC AML/ATF knowledge base |
| **milestone_1_fintrac_guidance_only.yaml** | 15KB | 33 | Only FINTRAC guidance | Already have acts/regs, adding guidance |
| **test_minimal.yaml** | 1.8KB | 3 | Minimal test set | Testing, development, demos |

## 1. manifest.yaml (Master Registry)

**Path:** `data/raw/manifest.yaml`

### Description
The master manifest containing ALL documents from all sources. This is automatically generated by merging other manifests.

### Structure
```yaml
# Merged manifest
# Generated: 2025-11-20T22:30:40.730273Z
# Merged from 2 sources

documents:
  - filename: S.C.1991, c. 46 - Bank Act.pdf
    category: act
    ...
  - filename: SOR-2002-184 - PCMLTFA Regulations.pdf
    category: regulation
    ...
  - filename: FINTRAC - Accountants Guidance.pdf
    category: guidance
    ...
```

### Contents
- **Acts:** Bank Act, PCMLTFA, PIPEDA
- **Regulations:** All related regulations
- **Guidance:** All FINTRAC guidance documents

### When to Use
```bash
# Production deployment with all documents
python3 batch_ingest.py data/raw/manifest.yaml --continue-on-error
```

### Auto-Generated
This manifest is created/updated by:
- `run_mapper_from_config.py` (downloads acts/regs)
- Merge operations (combining multiple manifests)

### Backups
Automatic backups created before each modification:
```
data/raw/manifest.backup.20251120_211551.yaml
data/raw/manifest.backup.20251120_212153.yaml
```

## 2. milestone_1_fintrac_pcmltfa.yaml (Full Framework)

**Path:** `data/raw/milestone_1_fintrac_pcmltfa.yaml`

### Description
Complete FINTRAC PCMLTFA compliance framework. This manifest contains everything needed to build a comprehensive AML/ATF knowledge base.

### Structure
```yaml
project:
  name: FINTRAC PCMLTFA Compliance - Milestone 1
  milestone: 1
  description: Complete AML/ATF regulatory framework for Canadian entities
  jurisdiction: federal
  regulator: FINTRAC
  total_documents: 36

documents:
  # 1 Act
  # 2 Regulations
  # 33 Guidance documents
```

### Contents (36 documents)

**Foundation (3 documents):**
1. **Act:** Proceeds of Crime (Money Laundering) and Terrorist Financing Act
2. **Regulation:** SOR-2002-184 (PCMLTFA Regulations)
3. **Regulation:** SOR-2001-317 (STR Regulations)

**Guidance (33 documents):**
- Sector-specific guidance (Accountants, Financial Entities, MSBs, etc.)
- Topic-specific guidance (Customer Due Diligence, Suspicious Transactions, etc.)
- Compliance guidance (Methods to Verify Identity, Record Keeping, etc.)

### When to Use

**Scenario 1: Fresh Start (No existing documents)**
```bash
# Process everything at once
python3 batch_ingest.py data/raw/milestone_1_fintrac_pcmltfa.yaml --continue-on-error
```

**Scenario 2: Phased Approach**
```bash
# Phase 1: Process acts and regulations first
python3 batch_ingest.py data/raw/milestone_1_fintrac_pcmltfa.yaml --continue-on-error
# (Skips guidance PDFs if they don't exist yet)

# Phase 2: Convert FINTRAC HTML to PDF
# ... manual conversion ...

# Phase 3: Re-run to process guidance
python3 batch_ingest.py data/raw/milestone_1_fintrac_pcmltfa.yaml --continue-on-error
# (Skips already-processed acts/regs, processes guidance)
```

### Priority Levels
- **Priority 1:** Acts (foundation)
- **Priority 2:** Regulations (implementing rules)
- **Priority 3:** Guidance (interpretation & compliance)

## 3. milestone_1_fintrac_guidance_only.yaml (Incremental)

**Path:** `data/raw/milestone_1_fintrac_guidance_only.yaml`

### Description
Contains ONLY the 33 FINTRAC guidance documents. Use this when you've already processed the acts and regulations, and just want to add guidance.

### Structure
```yaml
project:
  name: FINTRAC Guidance Only - Milestone 1
  milestone: 1-guidance
  description: FINTRAC guidance documents for Canadian AML/ATF compliance
  total_documents: 33

documents:
  # Only guidance documents, no acts or regulations
```

### Contents (33 documents)
All FINTRAC guidance documents:
- Accountants sector guidance
- Financial entities guidance
- Money services businesses guidance
- Customer due diligence methods
- Suspicious transaction reporting
- Record keeping requirements
- ... (27 more)

### When to Use

**Scenario: Already Have Acts/Regs, Adding Guidance**
```bash
# Skip validation (because parent acts aren't in THIS manifest)
python3 batch_ingest.py data/raw/milestone_1_fintrac_guidance_only.yaml \
  --skip-validation \
  --continue-on-error
```

**Why `--skip-validation`?**
The guidance documents reference `parent_act: "PCMLTFA"`, but that act isn't in this manifest. Validation would fail even though the parent act exists in storage from previous processing.

### Advantages
- **Saves Adobe Credits:** Only processes guidance (33 docs) instead of everything (36 docs)
- **Faster:** Skips acts/regulations if already processed
- **Incremental:** Add guidance to existing knowledge base

## 4. test_minimal.yaml (Development)

**Path:** `data/raw/test_minimal.yaml`

### Description
Minimal 3-document test set for development, testing, and demos. Covers all document types and segmentation methods.

### Structure
```yaml
project:
  name: "Minimal Test - 3 Documents"
  description: "Test bookmark segmenter with minimal Adobe credit usage"
  total_documents: 3

documents:
  # 1 Act (tests PDF bookmarks)
  # 1 Regulation (tests PDF bookmarks)
  # 1 Guidance (tests text pattern fallback)
```

### Contents (3 documents)
1. **Act:** PCMLTFA (425 sections) - Tests bookmark segmentation
2. **Regulation:** SOR-2002-184 (579 sections) - Tests bookmark segmentation
3. **Guidance:** FINTRAC Identity Methods (15 sections) - Tests pattern-based segmentation

### When to Use

**Development & Testing:**
```bash
# Test ingestion pipeline
python3 batch_ingest.py data/raw/test_minimal.yaml --continue-on-error

# Test embeddings generation
python3 generate_embeddings.py

# Test search
python3 search/semantic_search.py "customer due diligence"
```

### Advantages
- **Fast:** 3 documents process in ~10 minutes
- **Minimal Cost:** Uses minimal Adobe API credits
- **Representative:** Covers all document types and segmentation methods
- **Complete Pipeline:** Enough data to test Bronze → Silver → Gold → Search

### Perfect For
- Testing new features
- Debugging segmentation algorithms
- Demos and presentations
- CI/CD pipeline validation
- Quick iterations during development

## 🔄 Manifest Lifecycle

### Creation Flow
```
mapper_config.yaml
    ↓ (run_mapper_from_config.py)
manifest.yaml (generated)
    ↓ (manual curation)
milestone_1_fintrac_pcmltfa.yaml (focused subset)
    ↓ (extraction)
milestone_1_fintrac_guidance_only.yaml (guidance only)
    ↓ (minimal for testing)
test_minimal.yaml (3 docs)
```

### Merge Flow
```
milestone_1_fintrac_pcmltfa.yaml + other_manifest.yaml
    ↓ (restore_manifest.py merge)
manifest.yaml (merged master registry)
```

## 🛠️ Working with Manifests

### Validate Before Ingestion
```bash
python3 validate_manifest.py data/raw/manifest.yaml
python3 validate_manifest.py data/raw/milestone_1_fintrac_pcmltfa.yaml
python3 validate_manifest.py data/raw/test_minimal.yaml
```

### Process a Manifest
```bash
# With validation (recommended)
python3 batch_ingest.py data/raw/manifest.yaml --continue-on-error

# Skip validation (for incremental manifests)
python3 batch_ingest.py data/raw/milestone_1_fintrac_guidance_only.yaml \
  --skip-validation \
  --continue-on-error
```

### Merge Manifests
```bash
python3 restore_manifest.py merge \
  data/raw/milestone_1_fintrac_pcmltfa.yaml \
  data/raw/other_manifest.yaml \
  > data/raw/manifest.yaml
```

### List Backups
```bash
python3 restore_manifest.py list
```

### Restore from Backup
```bash
python3 restore_manifest.py restore 1
```

## 🔍 Troubleshooting

### Issue: Validation Fails for guidance_only Manifest

**Error:**
```
parent_act 'PCMLTFA' not found in manifest acts
```

**Solution:**
```bash
# Use --skip-validation flag
python3 batch_ingest.py data/raw/milestone_1_fintrac_guidance_only.yaml \
  --skip-validation \
  --continue-on-error
```

**Why:** The guidance-only manifest doesn't include the parent act (saves processing time), but validation checks for it.

---

### Issue: Which Manifest Should I Use?

**Decision Tree:**

1. **Testing/Development?** → `test_minimal.yaml`
2. **Need full FINTRAC framework?** → `milestone_1_fintrac_pcmltfa.yaml`
3. **Already have acts/regs, adding guidance?** → `milestone_1_fintrac_guidance_only.yaml`
4. **Production with all docs?** → `manifest.yaml`

# Regenerate from mapper config
